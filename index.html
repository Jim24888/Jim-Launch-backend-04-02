📌 转化归因管理机制

此处展示系统内已有的所有转化归因资产。

  • 由于业务需求，列表中隐藏了“媒体账号”及“深度强化目标ROI”列。
  • 点击上方的 【+ 新建转化归因模版】，可通过弹窗快速进行资产配置。

配置好的模板将沉淀在库中，供“上计划”时跨账号复用或直接应用。

📌 核心机制说明（必读）

  • 隐式自归因机制：为简化操作，系统在底层已将上报方式与“是否自归因”强绑定。
    • 选择 API，系统默认强制为 自归因（上报匹配后的数据）。
    • 选择 SDK，系统默认强制为 全量平台归因（上报全量数据由平台归因）。
  • 模板应用规则（上新计划时）：本页面仅保存配置为“模板”。在新建广告计划并应用此模板时，选择转化归因模版后，系统会统一提取模版参数，为当前账号直接新建转化归因（依赖API幂等性自动防重）。

📝 逐字段配置说明

一、 基础信息

  • 复用转化归因：功能开关。开启后可快速拉取已有配置，无需手动逐一填写。由于内部 API 与腾讯媒体端字段限制可能存在细微差异，拉取回填后请务必二次核对数据。
  • 复用账号 ID：当开启复用时必填。仅支持输入单个账号 ID。输入后需点击“获取转化列表”。
  • 选择转化归因：当成功获取转化列表后显示。选择特定的转化后，系统会自动将其参数映射至下方表单。
  • 转化名称：必填。用于在平台内标识该转化资产。最大长度限制为 60 个中文字或 120 个英文字符。

二、 转化场景与载体

  • 转化场景：必填。决定后续回传与归因链路的基础，目前支持：Android、iOS、鸿蒙 (Harmony) 及微信小游戏。
  • 应用 ID / 小游戏 ID：选填。营销载体的唯一标识。选择“微信小游戏”场景时填入小游戏 ID；其余场景填入应用 App ID。
  • 安卓渠道包 ID：选填。仅在场景为 Android 且使用 SDK 上报时显示并可用。格式必须严格为 0;具体渠道包ID，平台只会将自定义渠道包 ID 一致的数据计入该转化。

三、 数据上报与归因设置

  • 上报方式：必填。支持 API 与 SDK。
    注意：鸿蒙 (Harmony) 场景当前仅支持 API 上报方式。
  • 归因方式：必填。此选项受“转化场景”与“上报方式”的严格双重约束：
    • 场景为【微信小游戏】时：强制仅支持 注册 (REGISTER)。
    • 上报方式为【SDK】时：强制仅支持 激活 (ACTIVATION)。
    • APP 类场景 +【API】上报时：支持 点击 (CLICK)（若账户配置了白名单，也支持选择激活）。
  • 通过数据源 ID 创建：功能开关。仅在 SDK 上报（即全量平台归因）时显示。开启后，可手动指定特定的数据源。
  • 数据源 ID：条件必填。当开启“通过数据源 ID 创建”时必须填写。用于账户下存在多个数据源时，精准指定归因数据源。
  • 点击监测链接：条件必填。用于接收平台下发的点击及宏替换信息。当且仅当上报方式为 API（自归因），且场景为 Android 或 iOS 时，此项为必填，其他情况为选填或留空。

四、 优化目标设定

  • 浅度优化目标：必填。计划的核心优化事件（如激活、注册、次日留存、付费等）。
  • 深度优化维度：功能开关，支持“不使用”、“深度行为”、“深度 ROI”三选一。
  • 深度行为目标：选择“深度行为”后必填。如在浅度目标（激活）基础上，深度优化（付费次数）。
  • 深度 ROI 目标：选择“深度 ROI”后必填。如在浅度目标基础上，优化（首日付费 ROAS）。

🚀 上计划 - 模版应用机制

【转化归因模版 选择与搜索】

  • 下方的下拉框支持直接输入文字进行模糊搜索，体验更顺畅。

【套用逻辑（重点）】

  • 待一次上计划时，选择了该模版。若上计划时是对应账号，则直接引用该转化归因资产进行投放。
  • 若上计划时不是对应账号，则系统会使用此转化归因模版的内部配置，为该新账号全新创建一个转化归因资产。
  • 应用位置：草稿模版、上计划页（单账号与多账号）。
